Detailed Description

Behavior that responds to hazards by backing up the robot.

When hazards like obstacles are detected, this behavior commands the robot to back up a short distance to clear the hazard. It's typically used as a reactive safety mechanism within other behaviors.

Execute one cycle of the reflex behavior.

Execute one iteration of the reflex behavior.

Sends reverse velocity commands to back up the robot until the hazard is cleared, backup distance is reached, or a backup limit is hit.

Parameters
dataCurrent robot sensor and state data
Returns
State RUNNING if still backing up, SUCCESS if hazard cleared, FAILURE if hazard couldn't be cleared

This method handles the immediate response to detected hazards by backing up the robot a short distance. It monitors the traveled distance and checks if hazards are cleared or if backup limits are reached.

Parameters
dataCurrent sensor and state data from the robot
Returns
State Indicates the current execution state:
  • RUNNING: Backing up is still in progress
  • SUCCESS: Hazard has been cleared by backing up
  • FAILURE: Unable to clear hazard even after backing up
